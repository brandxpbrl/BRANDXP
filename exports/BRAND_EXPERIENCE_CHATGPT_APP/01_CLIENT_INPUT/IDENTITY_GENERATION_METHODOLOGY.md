# IDENTITY GENERATION METHODOLOGY — Brand Experience OS

## Propósito

Este archivo define la metodología oficial para crear `identity_cliente.md` o `identity.dm` dentro de Brand Experience OS.

Su función es permitir que cualquier proyecto, cliente o conversación en ChatGPT pueda generar una identidad de marca estructurada usando la lógica del sistema:

- marca como entidad
- identidad antes del diseño
- percepción antes de la venta
- storytelling como transformación
- universo visual como manifestación de identidad
- dirección comercial como consecuencia de claridad estratégica

Este archivo debe subirse a la carpeta del proyecto o usarse como contexto dentro de ChatGPT antes de crear la identidad de un cliente.

---

## Insumos mínimos

- Nombre del cliente
- Qué hace la marca
- Qué vende

- Notas o contexto general

## Insumos recomendados

- Instagram actual

# 3. Proceso general

## Paso 1 — Reunir contexto

Antes de escribir la identidad, ordenar la información disponible en estas categorías:

```text
Cliente
Oferta
Público
Problema
Percepción actual
Percepción deseada
Diferenciadores
Tono
Visuales
Storytelling
Contenido
Ventas
Próximos pasos
```

No inventar datos no provistos. Cuando falte información, marcarlo como:

```text
Dato no provisto
Supuesto estratégico
Recomendación a validar
```

## Paso 2 — Interpretar la marca como entidad

La marca no debe analizarse como una empresa solamente.

Debe interpretarse como una entidad con:

- energía
- voz
- percepción
- estética
- narrativa
- comportamiento
- promesa
- universo visual
- tensión de identidad

Pregunta central:

```text
¿Qué entidad de marca está intentando emerger detrás de este negocio?
```

## Paso 3 — Diagnosticar percepción

Analizar la diferencia entre:

```text
lo que la marca es
vs
lo que la marca transmite actualmente
```

Identificar:

- percepción actual
- percepción deseada
- brecha principal
- riesgos
- oportunidades
- señales de confianza
- señales de autoridad
- señales premium
- debilidades visuales o narrativas

## Paso 4 — Definir núcleo de marca

Construir el núcleo identitario:

- esencia
- propósito
- promesa
- personalidad
- valores
- diferenciadores
- territorio simbólico
- posicionamiento recomendado

La identidad debe responder:

```text
Por qué existe la marca.
Qué promete.
Qué la hace diferente.
Qué debe hacer sentir.
Qué lugar debe ocupar en la mente del público.
```

## Paso 5 — Definir público ideal

Describir al público desde lo emocional y comercial:

- quién es
- qué busca
- qué teme
- qué objeciones tiene
- qué necesita sentir para confiar
- qué lo mueve a comprar
- qué lenguaje entiende
- qué nivel de claridad necesita

## Paso 6 — Definir tono de voz

Crear una guía verbal práctica:

- personalidad verbal
- tono principal
- palabras que sí usa
- palabras que evita
- frases modelo
- tono para Instagram
- tono para WhatsApp
- tono comercial
- tono institucional
- tono emocional

El tono debe ser coherente con la entidad de marca.

## Paso 7 — Definir dirección visual

No diseñar todavía.

Primero definir dirección:

- atmósfera
- paleta sugerida
- significado de colores
- tipografías sugeridas
- composición
- fotografía
- iluminación
- texturas
- símbolos posibles
- estilo gráfico
- qué evitar visualmente

La regla principal:

```text
La estética debe transmitir la percepción deseada antes de decorar.
```

## Paso 8 — Definir sistema de logo

El sistema de logo debe ser una dirección conceptual, no necesariamente el diseño final.

Debe incluir:

- tipo de logo recomendado
- símbolo posible
- monograma posible
- versión principal
- versión horizontal
- versión compacta
- sello o badge
- uso en fondo claro
- uso en fondo oscuro
- qué evitar
- rutas creativas sugeridas

Evitar:

- símbolos obvios
- íconos genéricos
- logos de plantilla
- exceso de elementos
- baja legibilidad
- estética sin relación con la promesa

## Paso 9 — Definir storytelling

Crear la narrativa central de la marca:

- historia madre
- conflicto
- transformación
- mensaje madre
- manifiesto corto
- pilares narrativos
- claims
- frases memorables
- sistema de contenido emocional

El storytelling debe responder:

```text
Qué transformación promete esta marca.
Qué historia quiere que el público sienta.
Qué frase resume su mundo.
```

## Paso 10 — Definir contenido

Crear arquitectura de contenido:

- pilares
- formatos
- reels
- carruseles
- stories
- posts comerciales
- prueba social
- educación
- conversión
- calendario base
- CTA

El contenido debe convertir la identidad en presencia constante.

## Paso 11 — Definir dirección comercial

Transformar la identidad en venta:

- oferta principal
- categorías de servicio
- beneficios
- objeciones
- respuestas
- CTA
- WhatsApp flow
- propuesta comercial
- prueba de confianza

La venta no debe romper la identidad.

Debe sentirse como una extensión natural de la marca.

Usar este prompt dentro de ChatGPT junto con el contexto del cliente:

```text
Actúa como Brand Experience OS.

No respondas como asistente genérico.
No inventes datos no provistos.
Cuando falte información, marca “Dato no provisto” o “Recomendación a validar”.

Analiza la marca como una entidad viva, no solo como un negocio.

Usa estos principios:

- identidad antes de diseño
- percepción antes de venta
- marca como entidad
- storytelling como transformación
- estética como lenguaje emocional
- dirección visual como manifestación de identidad
- claridad comercial como extensión de la promesa

Genera el documento con esta estructura:

1. Instrucción inicial
2. Perfil del cliente

6. Público ideal
7. Tono de voz
```

# 7. Cómo usar este archivo en una carpeta de proyecto ChatGPT

Subir este archivo a la carpeta del proyecto como:

```text
IDENTITY_GENERATION_METHODOLOGY.md
```

Luego, para cada cliente, subir:

```text
MANUAL_FRAMEWORK_ANALYSIS.md
```

o pegar el contexto del cliente.

Después pedir:

```text
Usa IDENTITY_GENERATION_METHODOLOGY.md como metodología oficial.
Usa MANUAL_FRAMEWORK_ANALYSIS.md como contexto del cliente.
Genera identity_cliente.md listo para guardar en prompt_pack.
```
